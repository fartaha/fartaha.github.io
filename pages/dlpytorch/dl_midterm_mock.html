Q1. Why not using linear activation function (like y = x y = x y=xy = xy=x) or y = s i g n ( x ) y = s i g n ( x ) y=sign(x)y = sign(x)y=sign(x) in feedforward neural networks (FFNs)?

 Hint

https://iq.opengenus.org/linear-activation-function/
 Solution
  • Backprop requires differentiable functions with nonzero gradients in a meaningful portion of their domain. The derivative of the sign function is zero almost everywhere so weights will never update.
    [https://stats.stackexchange.com/questions/463488/why-is-the-sigmoid-activation-function-better-than-the-sign-function]
  • If linearities are used in every layer, effectively we end up with a linear (regression) model which is restricted in what it can represent (only linear relationships).
    [https://datascience.stackexchange.com/questions/111390/why-cant-we-use-linear-activation-function-in-hidden-layers]
